## 二、测试策略

### 2.1 单元测试

- 覆盖所有公共方法
- 使用 pytest 框架
- Mock 外部依赖（LLM API、文件系统、数据库）
- 目标覆盖率 >= 80%

### 2.2 集成测试

- 配置 → 日志 → LLM 客户端链路测试
- 数据库 CRUD 完整链路
- 工具注册 → 路由 → 执行链路

### 2.3 异常测试

- 配置缺失/错误
- LLM API 不可用
- 数据库损坏
- 文件权限异常
