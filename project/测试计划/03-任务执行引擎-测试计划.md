## 二、测试策略

### 2.1 单元测试

- 覆盖所有执行逻辑
- Mock 工具调用和 LLM 响应
- 目标覆盖率 >= 90%

### 2.2 集成测试

- 完整执行链路（接收 → 拆解 → 执行 → 判定）
- 使用真实工具（文件操作）

### 2.3 异常测试

- 工具执行失败
- LLM 响应异常
- 预算耗尽
- 超时终止
