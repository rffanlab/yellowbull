# 01-任务接收与步骤拆解 — 验收标准

> 对应任务文档：`01-task-reception-tasks.md`
> 模块范围：任务接收、意图识别、置信度评估、步骤拆解、质量校验、ContextStore、计划展示

---

## 一、验收目标

确保任务接收模块能够正确解析用户输入、评估任务置信度、将任务拆解为可执行的步骤序列，并对拆解质量进行校验。

---

## 二、验收标准

### T01-01: 任务接收与意图识别

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-01-01 | 简单任务识别 | "读取文件 xxx" 被识别为简单任务 | 输入简单指令，检查 Task 对象 |
| AC-01-01-02 | 复杂任务识别 | "重构整个模块" 被识别为复杂任务 | 输入复杂指令，检查 Task 对象 |
| AC-01-01-03 | 任务约束提取 | 输入中的约束条件被正确提取 | 输入含约束的任务，检查 constraints 字段 |
| AC-01-01-04 | 成功标准提取 | 输入中的成功标准被正确提取 | 输入含成功标准的任务 |
| AC-01-01-05 | 无效输入处理 | 空输入或无法解析的输入返回错误 | 输入无效内容 |

### T01-02: 置信度评估

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-02-01 | 高置信任务 | 明确具体的任务返回高置信度 | 输入明确任务，检查 confidence >= 0.8 |
| AC-01-02-02 | 中置信任务 | 模糊但可执行的任务返回中置信度 | 输入模糊任务，检查 0.4 <= confidence < 0.8 |
| AC-01-02-03 | 低置信任务 | 极度模糊或危险的任务返回低置信度 | 输入危险任务，检查 confidence < 0.4 |
| AC-01-02-04 | 置信度影响展示 | 低置信任务触发计划确认 | 输入低置信任务，检查是否展示计划 |

### T01-03: 步骤拆解

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-03-01 | 简单任务拆解 | 简单任务拆解为 1-3 步 | 输入简单任务，检查步骤数 |
| AC-01-03-02 | 复杂任务拆解 | 复杂任务拆解为 3-8 步 | 输入复杂任务，检查步骤数 |
| AC-01-03-03 | 工具分配 | 每步都有正确的 tool_hint | 检查每个 Step 的 tool_hint 字段 |
| AC-01-03-04 | 依赖关系 | 步骤间依赖关系正确标注 | 检查 depends_on 字段 |
| AC-01-03-05 | 关键步骤标记 | 关键步骤标记为 is_critical | 检查关键步骤的标记 |

### T01-04: Prompt 组装

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-04-01 | 7 层 Prompt 组装 | 拆解 Prompt 包含全部 7 层 | 检查生成的 Prompt 内容 |
| AC-01-04-02 | 工具清单注入 | 可用工具列表正确注入 | 检查 Prompt 中的工具清单 |
| AC-01-04-03 | 经验注入 | 有相关经验时注入 Prompt | 提供经验列表，检查 Prompt |
| AC-01-04-04 | Few-shot 示例 | 示例正确注入 | 检查 Prompt 中的示例 |

### T01-05: 质量校验

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-05-01 | 循环依赖检测 | 循环依赖被正确检测 | 构造循环依赖，验证检测 |
| AC-01-05-02 | 孤立步骤检测 | 孤立步骤被正确检测 | 构造孤立步骤，验证检测 |
| AC-01-05-03 | 工具可用性检查 | 无匹配工具时报告问题 | 使用不存在的 tool_hint |
| AC-01-05-04 | 校验失败修正 | 校验失败后触发 LLM 修正 | 构造无效拆解，验证修正流程 |

### T01-06: 步骤排序与合并

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-06-01 | 拓扑排序 | 依赖关系正确的步骤排在前面 | 检查排序结果 |
| AC-01-06-02 | 步骤合并 | 超过 max_steps 时合并步骤 | 构造超多步骤，验证合并 |
| AC-01-06-03 | 关键步骤保护 | 关键步骤不被合并 | 标记关键步骤，验证不被合并 |

### T01-07: ContextStore

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-07-01 | 数据存储 | 步骤结果正确存储 | store() 后 get() 验证 |
| AC-01-07-02 | 数据获取 | 按 step_id 正确获取 | 验证数据一致性 |
| AC-01-07-03 | 输入收集 | gather_inputs 正确收集上游输出 | 构造多步骤链路 |
| AC-01-07-04 | 缺失输入检测 | 缺失输入时抛出异常 | 构造缺失依赖 |

### T01-08: 拆解失败兜底

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-08-01 | JSON 错误重试 | JSON 格式错误重试 1 次 | Mock JSON 错误 |
| AC-01-08-02 | 降级为单步 | 校验修正失败降级为单步 | Mock 校验持续失败 |
| AC-01-08-03 | LLM 失败不降级 | LLM 调用失败不降级 | Mock LLM 异常 |

### T01-09: 计划展示与确认

| 编号 | 验收项 | 验收条件 | 验收方法 |
|------|--------|---------|---------|
| AC-01-09-01 | 计划渲染 | 步骤列表正确渲染 | 检查渲染输出 |
| AC-01-09-02 | 危险操作警告 | 危险操作正确警告 | 构造危险步骤 |
| AC-01-09-03 | 工具标签显示 | 工具类型标签正确显示 | 检查标签格式 |

## 三、验收通过条件

- [ ] 所有 AC 项 100% 通过
- [ ] 简单任务拆解准确率 >= 95%
- [ ] 复杂任务拆解合理率 >= 90%
- [ ] 质量校验正确率 100%
- [ ] 无 P0/P1 级别缺陷
