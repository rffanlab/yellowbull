# YellowBull (老黄牛)

本地开发 Agent：接任务 → 拆步骤 → 调工具 → 看结果 → 修正 → 记录经验

## 核心流程

```
用户输入任务
    │
    ▼
┌──────────────┐
│  1. 任务接收  │  ← 解析用户自然语言，明确任务目标
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 2. 经验回顾  │  ← 查经验库：类似任务有没有成功/失败记录？
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 3. 步骤拆解  │  ← 将任务拆成有序的子步骤
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 4. 执行循环  │  ← 逐步骤执行（核心循环）
│  ┌────────┐  │
│  │ 调工具 │  │
│  └───┬────┘  │
│      ▼      │
│  ┌────────┐  │
│  │ 看结果 │  │
│  └───┬────┘  │
│      ▼      │
│  ┌────────┐  │
│  │ 需修正? │──YES→ 回到调工具（重试/换策略）
│  └───┬────┘  │
│      │NO    │
│      ▼      │
│  下一步骤    │
└──────┬───────┘
       │ 全部完成
       ▼
┌──────────────┐
│ 5. 结果汇总  │  ← 汇总所有步骤结果，输出最终答案
└──────┬───────┘
       │
       ▼
┌──────────────┐
│ 6. 记录经验  │  ← 将本次执行的关键信息存入经验库
└──────────────┘
```

## 架构分层

| 层级 | 模块 | 说明 |
|------|------|------|
| CLI/UI 层 | `cli/` | 任务输入、结果展示、交互 |
| Agent 核心层 | `agent/` | 任务接收 → 步骤拆解 → 执行循环 |
| 支撑层 | `tools/` `experience/` | 工具调用、经验库 |
| 基础设施层 | `llm/` `config/` `storage/` | LLM 接口、配置管理、SQLite 持久化 |

## 功能模块

### 工具系统 (`tools/`)

- **FileTool** — 文件读写操作
- **ShellTool** — 执行 Shell 命令
- **CodeTool** — 代码分析相关操作
- **ToolRegistry** — 工具注册与分发

### 模型系统 (`models/`)

- **Task** — 任务对象，承载用户请求
- **SubTask / Step** — 子任务和步骤拆解
- **TaskResult / StepResult** — 执行结果
- **Experience** — 经验记录，用于回顾和学习

### LLM 接口 (`llm/`)

- **LLMClient** — 基于 LiteLLM 的多模型兼容客户端

### 配置管理 (`config/`)

- **Settings** — 基于 pydantic-settings 的配置加载

### 持久化 (`storage/`)

- **DatabaseManager** — 基于 aiosqlite 的异步 SQLite 存储

## 快速开始

```bash
# 安装依赖
pip install -e ".[dev]"

# 运行测试
pytest -v

# 使用 CLI
yellowbull "你的任务描述"
```

## 技术栈

- **Python >= 3.10**
- **LangGraph** — Agent 工作流编排
- **LiteLLM** — 统一 LLM 接口
- **Pydantic** — 数据模型与配置校验
- **Click + Rich** — CLI 界面
- **aiosqlite** — 异步 SQLite 存储
- **pytest** — 测试框架
